# Exception Handling Fixes — 2026.03.07

Audit of unhandled exceptions in TAM source code, triggered by investigating a VS Code OOM event (149 GB memory usage). TAM was ruled out as the cause (the OOM was due to many open editor tabs), but three exception-handling issues were identified and fixed.

## Fixes

### 1. Unhandled promise rejection in `activityTracker.ts`

**File:** `src/activityTracker.ts:170`

**Problem:** `terminal.processId` returns a VS Code `Thenable`. The `.then()` handler had no rejection handler. If a terminal is disposed before its PID resolves, this produces an unhandled promise rejection.

**Fix:** Wrapped in `Promise.resolve()` (since `Thenable` lacks `.catch()`) and added a `.catch()` handler.

```ts
// Before
terminal.processId.then((pid) => { ... });

// After
Promise.resolve(terminal.processId).then((pid) => { ... }).catch(() => {
  // Terminal may be disposed before processId resolves
});
```

### 2. Unguarded filesystem calls in `treeView.ts`

**File:** `src/treeView.ts:195-210`

**Problem:** `buildTerminalItem()` calls six `has*()` functions (`hasClaudeMd`, `hasMemoryMd`, `hasClaudeRules`, `hasClaudeCommands`, `hasProjectSettings`, `hasProjectAgents`) that perform `fs.existsSync()` / `fs.readdirSync()` operations. While some inner functions had try/catch, the outer call site did not. An inaccessible path (permissions, broken symlink) could crash the tree view render cycle.

**Fix:** Wrapped all six calls in a single try/catch block with defaults of `false`.

### 3. Unbounded file read in `claudeHistoryCache.ts`

**File:** `src/claudeHistoryCache.ts:33`

**Problem:** `refreshCacheIfNeeded()` called `fs.readFileSync(HISTORY_FILE, "utf-8")` on the entire `~/.claude/history.jsonl` file with no size limit. As history grows, this causes proportional memory allocation spikes every time the file's mtime changes.

**Fix:** Added a 256KB tail-read limit. Files smaller than 256KB are read in full. Larger files use `fs.openSync` + `fs.readSync` to read only the last 256KB, skipping the first partial line. This matches the pattern already used in `claudeContextUsage.ts`.

## Severity Assessment

| # | File | Severity | Impact |
|---|------|----------|--------|
| 1 | `activityTracker.ts` | Medium | Unhandled rejection errors in dev console; could trigger VS Code error telemetry |
| 2 | `treeView.ts` | Low | Tree view crash on inaccessible paths (rare edge case) |
| 3 | `claudeHistoryCache.ts` | Low | Memory spikes proportional to history file size (not a leak, but wasteful) |
