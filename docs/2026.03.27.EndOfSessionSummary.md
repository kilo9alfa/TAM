## Executive Summary

Two features were implemented this session:

1. **"Open in Finder" right-click menu** — New context menu item on all terminals (local and remote) in the TAM sidebar. Right-click a terminal → "Open in Finder" opens its CWD in macOS Finder. Uses the terminal's resolved CWD (from `claudeInfo.cwd` or `record.cwd`).

2. **Model-aware context percentage** (uncommitted) — `claudeContextUsage.ts` was updated to detect the model from the JSONL session file and use the correct context window size: 1M tokens for Opus 4.x / Sonnet 4.x, 200K for Haiku 4.5 and older models. Previously hardcoded to 200K, which made Opus/Sonnet sessions show inflated context percentages.

**Current Status:** Both features built and installed. Changes are **uncommitted** on `main`.

## Important Decisions

**Decision:** Enable "Open in Finder" for all terminals including remote (`viewItem =~ /^terminal/`)
**Rationale:** Remote terminals may have local CWDs if they're SSH-forwarded or local processes. If the CWD doesn't exist locally, `open` will simply fail silently — no harm done.

**Decision:** Detect model from JSONL session file for context window sizing
**Rationale:** Claude Code now supports 1M context for Opus/Sonnet 4.x. Hardcoding 200K inflated the percentage display by 5x for these models. Reading the model from the session data is the only reliable way to determine the correct denominator.
